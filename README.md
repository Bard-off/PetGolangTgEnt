# PetGolangTgEnt

Проект Telegram-бота на языке Go, демонстрирующий работу с ORM [ent](https://entgo.io/) и базой данных PostgreSQL.

## 🛠 Технологии

* **Язык:** Go (1.23+)
* **Бот-фреймворк:** [go-telegram/bot](https://github.com/go-telegram/bot)
* **ORM:** [ent](https://entgo.io/)
* **База данных:** PostgreSQL
* **Конфигурация:** `godotenv` (загрузка переменных окружения из `.env`)

## ✨ Функционал

* **Автоматические миграции:** При запуске приложение автоматически синхронизирует схему базы данных.
* **Регистрация пользователей:** При получении сообщения бот автоматически сохраняет Telegram ID и имя пользователя в базу данных.
* **Связи данных:** Демонстрация связей "Один ко многим" (User -> Posts).
* **Команды:**
    * `/get` — получает информацию о текущем пользователе и список его постов из базы.
* **Конкурентность:** Использование каналов и `sync.WaitGroup` для безопасной обработки запросов к БД.

## 🚀 Быстрый старт

### 1. Подготовка окружения
Убедитесь, что у вас установлен Go и запущен PostgreSQL. Создайте в корне проекта файл `.env` и добавьте в него ваши данные:

```env
BOT_KEY=ваш_токен_от_BotFather
DB_URL=postgres://user:password@localhost:5432/dbname?sslmode=disable
```
### 2. Установка зависимостей

```Bash
go mod download
```
### 3. Генерация кода Ent
Если вы вносили изменения в файлы папки `ent/schema/`, выполните генерацию:

```Bash
go generate ./ent
```

### 4. Запуск приложения

```Bash
go run cmd/app/main.go
```

## 📂 Структура проекта

* `cmd/app/main.go` — точка входа, инициализация клиента БД и Telegram-бота.
* `ent/` — сгенерированный код ORM и схемы данных.
* `ent/schema/` — описание таблиц User и Post.
* `internal/config/` — логика загрузки конфигурации и маппинг переменных окружения.
* `internal/repository/` — функции для работы с данными (CRUD-операции).


## 📝 Особенности реализации
* В проекте используется паттерн ограничения количества одновременных запросов к БД через буферизированный канал (семафор) в `main.go`.
* Обработка ошибок Telegram API и базы данных выводится в консоль для удобства отладки.
